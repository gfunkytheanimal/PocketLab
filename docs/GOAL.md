# Pocket Universe Lab Goal

Build a browser physics toybox where the first screen is a dark empty vacuum and every dropped object makes invisible physics visible.

The north star is not scientific purity. The north star is cause and effect that feels alive: two objects should be entertaining enough to watch on their own, while larger systems should turn into beautiful chaos.

## Current Design Rules

- Keep the physics engine stable, simple, and tunable.
- Prefer procedural visual systems around objects instead of relying only on static models.
- Make every force readable: heat, gravity, charge, tidal stress, spacetime folding, collisions, and fields should show up visually.
- Add controls in-app before hardcoding values repeatedly.
- Use CC0 or properly licensed assets wherever possible.

## High-Value Backlog

- More pair-specific reactions: planet-moon capture, star-gas ignition, black-hole/wormhole jets, magnet dust auroras, UFO tractor behavior.
- Better particles and fluids: smooth gas, nebula brushes, dust motes, plasma, steam, molten sprays, debris fields.
- Dev mode expansion: tune shaders, particle counts, reaction strength, camera, and physics presets live.
- More authored CC0 assets: UFO, alien, light beam, rocks, vehicles, oddities.
- Save/load playground states and shareable scenarios.
- Scenario presets that feel like toys: Contact, Alien Lab, Moon Capture, Quasar Jet, Ring World, Fluid Nebula, Solar Harmonic.
- Optional heavy-GPU mode for volumetric fields, self-shadowing particles, and higher-quality bloom.
